# AnimeBaza — Use-Case Specification

**Проект:** Веб-приложение-каталог аниме «AnimeBaza» (учебная практика УП.02).
**Назначение:** Поиск и просмотр информации об аниме, ведение личных списков просмотра.
**Данные:** Загружаются через Shikimori GraphQL API (публичные данные). Пользовательские списки хранятся локально в браузере.
**Страницы:** каталог `/`, детальная страница тайтла `/anime/[id]`, псевдо-вход `/login`, профиль `/profile`.

> Этот файл предназначен для передачи нейросети, чтобы построить use-case диаграмму.
> Все сценарии описаны функционально, без привязки к конкретной реализации.
>

## Акторы

| Актор | Описание |
|-------|----------|
| **Посетитель** | Неавторизованный пользователь. Может просматривать каталог, страницы тайтлов, искать аниме |
| **Пользователь** | Авторизованный пользователь (псевдо-логин). Может всё, что посетитель, плюс управлять списками и оценками |

**Отношение обобщения:** Пользователь наследует все возможности Посетителя (hollow triangle arrow от Пользователя к Посетителю).

## Варианты использования — сводка

**Посетитель:**
1. Просмотр каталога ← «extend» — Фильтрация каталога
2. Просмотр каталога ← «extend» — Поиск аниме
3. Просмотр деталей тайтла
4. Псевдо-авторизация

**Пользователь** (всё что у Посетителя, дополнительно):
5. Управление списками
6. Оценка аниме
7. Просмотр профиля

**Всего:** 2 актора, 7 вариантов использования (+ 2 extend-расширения)

## Детальные сценарии

### UC-1: Просмотр каталога

- **Акторы:** Посетитель, Пользователь
- **Триггер:** Открытие главной страницы
- **Основной поток:**
  1. Система загружает список аниме
  2. Система отображает карточки: постер, название, тип (TV/Movie/OVA), год, иконку статуса просмотра
  3. Пользователь наводит курсор на карточку — через небольшую задержку появляется попап с деталями (постер, название, статус выхода, рейтинг, жанры, описание, кнопка добавления в список)
  4. При уводе мыши попап скрывается с небольшой задержкой
  5. Пользователь может применить фильтры: тип, статус (онгоинг/вышел/анонс), сезон, сортировка
  6. Пользователь может выполнить поиск по названию (с debounce или по Enter)
  7. Система подгружает следующие страницы результатов по кнопке «Загрузить ещё»
- **Альтернативный поток (нет результатов):** Сообщение «Ничего не найдено»
- **Альтернативный поток (ошибка API):** Сообщение с кнопкой «Повторить»

### UC-2: Просмотр деталей тайтла

- **Акторы:** Посетитель, Пользователь
- **Триггер:** Клик на карточку в каталоге или переход по ссылке `/anime/123`
- **Основной поток:**
  1. Система загружает полную информацию об аниме
  2. Система отображает: большой постер, название (русское + оригинальное), тип, статус выхода, количество эпизодов, рейтинг, жанры
  3. Система отображает описание аниме
  4. Система отображает плеер-заглушку (постер + иконка play + сообщение что видео недоступно)
  5. Если пользователь авторизован, система показывает блок добавления в список и блок оценки
- **Альтернативный поток (ошибка):** Сообщение с кнопкой «Повторить»

### UC-3: Управление списками

- **Актор:** Пользователь
- **Триггер:** Нажатие кнопки «В список» в каталоге, попапе или на странице тайтла
- **Основной поток:**
  1. Пользователь нажимает «В список»
  2. Система добавляет аниме в список со статусом «Запланировано»
  3. Кнопка заменяется на индикатор текущего статуса
- **Альтернативный поток (смена статуса):**
  1. Пользователь выбирает другой статус: Смотрю / Просмотрено / Отложено / Брошено
  2. Система обновляет статус
  3. Статус синхронизируется на всех страницах
- **Альтернативный поток (удаление):**
  1. Пользователь нажимает «Удалить»
  2. Система удаляет аниме из всех списков

### UC-4: Просмотр профиля и списков

- **Актор:** Пользователь
- **Триггер:** Переход на `/profile`
- **Основной поток:**
  1. Система отображает имя пользователя и статистику по 5 спискам (planned/watching/completed/on_hold/dropped)
  2. Система отображает 5 секций — по одной на каждый статус
  3. Каждая секция сворачивается/разворачивается, показывает заголовок со счётчиком
  4. Внутри секции — список аниме с мини-постером, названием, рейтингом, типом
  5. При наведении на элемент списка — попап с деталями аниме
  6. Пользователь может удалить аниме из списка
- **Альтернативный поток (пустой список):** Сообщение «Здесь пока ничего нет» + ссылка на каталог
- **Альтернативный поток (выход):** Пользователь выходит — система очищает данные

### UC-5: Псевдо-авторизация

- **Актор:** Посетитель
- **Триггер:** Переход на `/login`
- **Основной поток:**
  1. Посетитель вводит имя пользователя (минимум 2 символа)
  2. Посетитель нажимает «Войти»
  3. Система проверяет что имя не пустое, не короче 2 символов
  4. Система сохраняет имя
  5. Система перенаправляет на профиль
- **Альтернативный поток (короткое имя):** Кнопка входа заблокирована, пока имя не достигнет 2 символов

### UC-6: Оценка аниме

- **Актор:** Пользователь
- **Предусловие:** Аниме добавлено в список пользователя
- **Триггер:** Выбор рейтинга на странице тайтла (звёзды 1–10)
- **Основной поток:**
  1. Пользователь выбирает оценку (1–10)
  2. Система сохраняет оценку
  3. Оценка сразу отображается на странице тайтла и в профиле
- **Альтернативный поток (повторная оценка):** Пользователь выбирает другое значение — система обновляет

## Инструкция для построения диаграммы

1. **Акторы:** изобразите двух человечков (Посетитель слева, Пользователь справа). Соедините их стрелкой с полым треугольником от Пользователя к Посетителю — отношение обобщения (Пользователь наследует возможности Посетителя).

2. **Граница системы:** прямоугольник «AnimeBaza», содержащий все варианты использования.

3. **Варианты использования (эллипсы внутри границы):**
   - UC-1: Просмотр каталога
   - Фильтрация каталога (extend → UC-1, пунктирная стрелка с «extend»)
   - Поиск аниме (extend → UC-1, пунктирная стрелка с «extend»)
   - UC-2: Просмотр деталей тайтла
   - UC-5: Псевдо-авторизация
   - UC-3: Управление списками
   - UC-6: Оценка аниме
   - UC-4: Просмотр профиля

4. **Связи ассоциации (сплошные линии):**
   - Посетитель → Просмотр каталога, Просмотр деталей тайтла, Псевдо-авторизация
   - Пользователь дополнительно → Управление списками, Оценка аниме, Просмотр профиля
   - (Use case'ы Фильтрация и Поиск наследуются через extend от UC-1)

5. **Отношения:**
   - UC-1 ← «extend» — Фильтрация каталога (пунктир, стрелка на UC-1)
   - UC-1 ← «extend» — Поиск аниме (пунктир, стрелка на UC-1)
