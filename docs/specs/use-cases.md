# Use-Case Specification

## Акторы

| Актор | Описание |
|-------|----------|
| **Посетитель** | Неавторизованный пользователь. Может просматривать каталог, страницы тайтлов, искать аниме |
| **Пользователь** | Авторизованный пользователь (псевдо-логин). Может всё, что посетитель, плюс управлять списками и оценками |

## Диаграмма вариантов использования

Будет добавлена после выполнения Задания 2.

## Сценарии

### UC-1: Просмотр каталога

- **Актор:** Посетитель, Пользователь
- **Предусловие:** Открыта главная страница
- **Основной поток:**
  1. Система загружает список аниме (топ по рейтингу) через Shikimori GraphQL
  2. Система отображает карточки с постером, названием, типом, рейтингом
  3. Пользователь может применить фильтры (тип, статус, сезон, сортировка)
  4. Пользователь может ввести поисковый запрос
  5. Система обновляет список при изменении фильтров
  6. Пользователь может переключать страницы через пагинатор
- **Постусловие:** Отображён соответствующий набор аниме

### UC-2: Просмотр деталей тайтла

- **Актор:** Посетитель, Пользователь
- **Предусловие:** Пользователь кликнул на карточку или перешёл по ссылке `/anime/[id]`
- **Основной поток:**
  1. Система загружает полную информацию об аниме через Shikimori GraphQL
  2. Отображаются: постер, название, тип, статус, рейтинг, жанры, описание, студия
  3. Отображается плеер-заглушка (постер + иконка play + «Видео временно недоступно»)
- **Постусловие:** Детальная страница с информацией о тайтле

### UC-3: Добавление аниме в список

- **Актор:** Пользователь
- **Предусловие:** Пользователь авторизован, открыт каталог или страница тайтла
- **Основной поток:**
  1. Пользователь нажимает «В список» на карточке или на странице тайтла
  2. Система добавляет аниме в localStorage со статусом «Запланировано»
  3. Кнопка меняется на индикатор статуса + кнопку удаления
- **Альтернативный поток:**
  1. Пользователь выбирает другой статус из 5 доступных (planned/watching/completed/on_hold/dropped)
  2. Система обновляет статус в localStorage
- **Постусловие:** Аниме сохранено в списке пользователя

### UC-4: Просмотр профиля и списков

- **Актор:** Пользователь
- **Предусловие:** Пользователь авторизован
- **Основной поток:**
  1. Пользователь переходит на `/profile`
  2. Система отображает имя пользователя и статистику по 5 спискам
  3. Пользователь может переключаться между вкладками списков
  4. Каждая вкладка показывает карточки аниме с постером, названием и статусом
  5. Пользователь может удалить аниме из списка
- **Альтернативный поток (пустой список):** Отображается сообщение «Nothing here yet» + ссылка на каталог
- **Постусловие:** Пользователь видит свои списки

### UC-5: Псевдо-авторизация

- **Актор:** Посетитель
- **Предусловие:** Посетитель не авторизован
- **Основной поток:**
  1. Посетитель переходит на `/login`
  2. Вводит имя пользователя (минимум 2 символа)
  3. Нажимает «Войти»
  4. Система сохраняет имя в localStorage и перенаправляет на `/profile`
- **Постусловие:** Посетитель становится Пользователем
